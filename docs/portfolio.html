Project 1: Senior Design Project sdmay26-26, CCC (C Code Critiquer)

Team Project

My Role: Process and Inquiry Manager, Flex Developer

As a team of four, we work to refine an existing prototype and implement a comprehensive software solution designed foremost to analyze C code for CPR E 2880 but with a focus on future extensibility to other programming languages and courses. The critiquer platform enables instructors to upload code snippets and assign known antipatterns which the system can automatically detect and helpfuly warn in student code thereafter. The program uses a sleek web interface for instructor management and student usage and a robust backend and database solution to store, detect, and report upon code issues.

Our goal from the beginning was to bring into fruition the existing application prototype so that CPR E 2880 students could finally begin to avail its benefits. From a high level, we planned to close out several items:

  1. Correct existing bugs in the backend controller code and undertake a general rearchitecting of the inherited codebase.
  2. Refactor and extend the existing Regex and XPath detection logic to catch bugs more effectively; add more antipatterns corresponding to common issues with student code.
  3. Refine the user interface to improve usability and enable fine-grained access control and management tools for different user types.
  4. Write unit and integration tests to ensure both formal and practical correctness.
  5. Integrate MSAL authentication to improve security and promote commonality in management through AD-backed authorization.

With a smaller team, we communicated appropriate expectations with our advisor and have carefully limited project scope to ensure we end up with a workable product come May 2026. For the first semester—4910—we plan to primarily make updates and tweaks to the existing system to prepare for new features we will implement during the second—4920.

My Contributions (Big Picture)

I serve as the primary point-of-contact with our advisor, Dr. Rover, and our other client and collaborator, James Joseph. I lead team contributions to ensure timely completion and submission of all project deliverables and head new lines of inquiry at the front of our project to develop and refine ideas to drive our project forward. On the day-to-day, I serve as a flex developer owing to my wide range of past experience with full-stack and embedded-systems programming, and I contribute wherever necessary. I have worked to investigate antipattern detection logic and interface usability, and I plan to develop improved database schema and backend architecture to enable the necessary divisions by lab section and management tools for TAs and instructors alike thereupon.

Project 2: Ray Tracer for COM S 3360

Individual Project

My Role: Lead Developer (all development)

Based on phyiscal principles of light travel and optics, my ray tracer, Irradiance produces photorealistic renders using a performance-optimized architecture implementing parallelization and the bounding volume hierarchy acceleration structure. Irradiance accurately models light transmission through diffuse, dielectric, metallic, and colloidal surfaces and volumes and employs the GGX microfacet distribution with the Cook-Torrence specular model over realistic BRDF materials and HDR tonemapping with a skybox for phyiscally accurate and visually appealing results. Below are depicted a few early renders from the program.

Relevant Skills: C++, OpenGL, GLM, OLCPGE, linear algebra, Monte Carlo path tracing, computer graphics,

Resources Used: Ray Tracing in One Weekend, Ray Tracing The Next Week, Ray Tracing The Rest of Your Life, lecture material, assorted online resources.

Project 2: Search and Rescue CyBot Project for CPR E 2880

Team Project

My Role: Lead Developer (all software development)

With a mock userbase of the U.S. Coast Guard, our CyBot project was designed to successfully integrate various sensors, including an IR transmitter, an ultrasonic pinger, motor encoder measurements for distance calibration, bump and cliff sensors for near-field object detection, and a BNO055 IMU for accurate global positioning. The CyBot could successfully navigate a complex obstacle course to recover lost assets from a mock shipwreck using onboard data and a human-readable user-interface desktop application. An example navigable obstacle course and object scan on the desktop applicatio interface are shown below.

Relevant Skills: C, Python, Tkinter, I2C, UART, Tiva MCU.

Resources Used: TivaWare datasheet, CyBot API documentation, lecture material, provided laboratory material, assorted online resources.
